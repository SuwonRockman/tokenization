# 토크나이제이션 (Tokenization) - LLM 기초

## 학습 목표

LLM이 텍스트를 어떻게 이해하는지의 첫 단계인 **토크나이제이션**을 깊이 있게 학습합니다.

- **토크나이제이션이란?**: 텍스트를 작은 단위(토큰)로 나누는 과정
- **왜 필요한가?**: LLM은 토큰 단위로 텍스트를 처리
- **어떻게 작동하는가?**: BPE, WordPiece, SentencePiece 등 다양한 방식

## 핵심 개념

### 1. 토큰 (Token)
- 텍스트의 최소 단위 (단어, 음절, 문자 등)
- LLM은 각 토큰을 숫자 ID로 변환하여 처리

### 2. 토크나이저 (Tokenizer)
- 텍스트 → 토큰 ID 변환하는 도구
- 모델마다 다른 토크나이저 사용 (GPT, BERT, 등)

### 3. 주요 토크나이제이션 방식

| 방식 | 설명 | 사용 모델 |
|------|------|----------|
| **BPE** (Byte Pair Encoding) | 자주 나오는 문자 쌍을 반복 합치기 | GPT-2, GPT-3 |
| **WordPiece** | 단어를 서브워드 단위로 분할 | BERT, RoBERTa |
| **SentencePiece** | 언어 구분 없이 동작 (다국어) | mBART, mT5 |

## 학습 흐름 (5단계)

### 1️⃣ 토크나이제이션이 왜 필요한가?
- LLM이 텍스트를 이해하는 방식
- 토크나이제이션 없이는 불가능한 이유
- 실무 사례 (ChatGPT, BERT 등)

### 2️⃣ 어떤 방식들이 있고, 각각의 장단점은?
- **BPE** (Byte Pair Encoding) - GPT 계열
- **WordPiece** - BERT 계열  
- **SentencePiece** - 다국어 (mBART, mT5)
- 각 방식의 트레이드오프 분석

### 3️⃣ 한글/영문/다국어 토크나이제이션의 차이
- 언어별 특성과 토크나이제이션 방식의 차이
- 왜 한글은 더 복잡한가?
- 다국어 모델의 해결 방식

### 4️⃣ 실제 토크나이제이션 활동 ⭐ [핵심]
- 같은 텍스트를 여러 토크나이저로 실행
- 결과 비교 분석 (토큰 개수, 길이, 효율성)
- 왜 차이가 나는지 분석
- 기획자의 관점: 어떤 토크나이저를 선택할 것인가?

### 5️⃣ LLM 선택 시 토크나이제이션 고려사항
- 토크나이제이션 성능이 모델 선택에 미치는 영향
- 기획자의 의사결정 포인트
- 실무에서 고려해야 할 지표들
